常见陷阱

在C++中，如果y非0，那么总会返回true，但是在Java中，编译器会试图将这个int型数据转换为应接收的布尔类型数据，最后会发生编译期错误。

类型转换

若将数据类型进⾏“向下转换”（Narrowing Conversion）的操作（将容量较⼤的数据类型转换成容量较⼩的类 型），可能会发⽣信息丢失的危险。此时，编译器会强迫我们进⾏转型， 好⽐在提醒我们：该操作可能危险，若你坚持让我这么做，那么对不起， 请明确需要转换的类型。 对于“向上转换”（Widening conversion），则 不必进⾏显式的类型转换，因为较⼤类型的数据肯定能容纳较⼩类型的数 据，不会造成任何信息的丢失。

除了布尔类型的数据，Java允许任何基本类型的数据转换为另一种基本类型的数据。

类型提升

对⼩于 int 的基本数据类型（即 char、byte 或short）执⾏任何算术或按位操作，这些值会在执⾏操作之前类型提升为int，并且结果值的类型为 int。若想重新使⽤较⼩的类型，必须使⽤强制转换（由于重新分配回⼀个较⼩的类型，结果可能会丢失精度）。通常，表达式中最⼤的数据类型是决定表达式结果的数据类型。float 型和double 型相乘，结果是 double 型的；int 和 long 相加，结果是 long型。
